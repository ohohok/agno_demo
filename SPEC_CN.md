## 🔧 技术决策

### 决策 1: 不使用数据库
**理由**: 
- 保持服务轻量级
- Agno 框架管理会话状态
- 更容易部署和维护
- 如需要可以稍后添加

**权衡**:
- ✅ 优点: 简单、快速、无需数据库维护
- ❌ 缺点: 重启后会话丢失、无持久化

---

### 决策 2: OpenAI 兼容接口
**理由**:
- 智谱 AI 提供 OpenAI 兼容的 API
- 利用现有的 OpenAI SDK
- 易于在提供商之间切换
- 文档完善且经过测试

**权衡**:
- ✅ 优点: 灵活、良好支持、易于集成
- ❌ 缺点: 需要 `openai` 包依赖

---

### 决策 3: 内存会话存储
**理由**:
- Agno 在内部处理会话管理
- 基本用例不需要外部存储
- 更快的响应时间

**权衡**:
- ✅ 优点: 快速、简单、无外部依赖
- ❌ 缺点: 不持久、内存随会话增长

## 🐛 已知问题

### 问题 1: 会话持久化
**严重程度**: 低  
**描述**: 服务器重启时会话丢失  
**临时方案**: 客户端应用应管理会话 ID  
**修复**: 如需要持久化，添加数据库支持（规格 8）

### 问题 2: 无限速保护
**严重程度**: 中  
**描述**: 没有防止 API 滥用的保护  
**临时方案**: 使用反向代理（nginx）进行速率限制  
**修复**: 实现中间件（规格 6）

### 问题 3: 无身份验证
**严重程度**: 高（生产环境）  
**描述**: API 对知道端点的任何人开放  
**临时方案**: 部署在 VPN 或内部网络后面  
**修复**: 添加 API key 认证（规格 6）
